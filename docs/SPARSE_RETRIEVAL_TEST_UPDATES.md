在 `tests/test_backend_integration.py` 中添加了完整的 BGE-M3 稀疏检索测试能力和详细日志。

**位置：** `run_all_tests()` 方法开始处

**功能：**
- 在测试开始时检查 BGE-M3 稀疏编码器状态

## 6. 后端服务增强

**文件：** `backend/app/modules/ingestion/service.py`

**功能：**
- 在 `_vectorize_text_chunks()` 中添加稀疏向量统计信息
- 统计成功/失败的chunks数量
- 计算平均稀疏向量维度
- 在返回结果中包含 `sparse_stats` 字段

**统计信息包括：**
- `total_chunks`: 总chunks数量
- `successful`: 成功生成稀疏向量的chunks数量
- `failed`: 失败的chunks数量
- `total_sparse_elements`: 总非零元素数量
- `avg_sparse_size`: 平均稀疏向量维度

## 测试运行方式

运行完整测试：
```bash
cd tests
python test_backend_integration.py
```

测试将自动：
1. 检查 BGE-M3 编码器状态
2. 测试文档解析
3. 测试图片解析
4. 测试稀疏检索功能
5. 测试完整检索流程（包含稀疏检索）

## 预期输出

测试成功时，你会看到：
- ✅ BGE-M3 编码器健康检查通过
- ✅ 文档入库时稀疏向量生成成功
- ✅ 检索时稀疏检索结果正常返回
- ✅ 详细的稀疏向量统计信息
- ✅ 各检索类型的结果数量统计

## 注意事项

1. **首次运行**：BGE-M3 模型首次加载需要下载（约 1-2GB），可能需要一些时间
2. **模型加载**：如果模型未加载，测试会显示警告但不会失败
3. **数据依赖**：稀疏检索测试需要先有数据入库，建议先运行完整检索流程测试
4. **Float16 配置**：根据用户修改，当前使用 Float32（use_fp16=False），模型大小约 2.27GB

## 验证点

测试会验证：
- ✅ BGE-M3 编码器可以正常初始化
- ✅ 查询可以成功编码为稀疏向量
- ✅ 文档可以成功编码为稀疏向量
- ✅ 稀疏检索可以返回结果
- ✅ 稀疏向量与密集向量可以同时存储和检索
- ✅ RRF 融合可以正确处理稀疏检索结果
