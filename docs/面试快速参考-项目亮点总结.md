# Multi-Modal 智能路由可扩展知识库 RAG Agent - 面试快速参考

> **一句话总结**：面向多知识库、多模态场景的企业级 RAG 系统，具备智能路由、混合检索、可视化思考链等核心能力。

## 🎯 项目核心价值

**解决的问题**：
- 企业拥有多个异构知识库（技术文档、产品手册、图片资料等）
- 用户查询需要智能选择最相关的知识库
- 支持文档和图像的统一检索与回答生成
- 需要可解释的 AI 决策过程

**核心能力**：
- ✅ 多模态数据处理（PDF、文档、图片）
- ✅ 智能路由系统（基于知识库画像）
- ✅ 混合检索策略（Dense + Sparse + Visual）
- ✅ 可视化思考过程（实时展示 AI 决策步骤）
- ✅ 深度 RAG 调试（完整检索链路调试信息）

## 💡 六大核心创新点

### 1. 知识库画像动态生成 ⭐⭐⭐⭐⭐

**创新**：K-Means 聚类 + LLM 主题摘要的混合方法

**实现**：
- 向量采样（数据量 < 5000 全量，≥ 5000 蓄水池采样）
- K-Means 聚类（K = sqrt(N/2)）
- LLM 生成主题摘要并向量化
- 存储到 kb_portraits collection

**效果**：知识库选取准确性与稳定性显著提升

**面试话术**：
> "我创新性地采用 K-Means 聚类 + LLM 主题摘要的混合方法，自动分析知识库内容特征并生成结构化画像。通过向量采样、聚类分析和主题摘要生成，解决了多知识库场景下的智能路由难题。"

### 2. 三路融合混合检索 ⭐⭐⭐⭐⭐

**创新**：Dense + Sparse + Visual 三路并行检索与 RRF 融合

**实现**：
- **Dense**：语义向量检索（BGE Embedding，4096 维）
- **Sparse**：稀疏向量检索（BGE-M3，关键词精确匹配）
- **Visual**：视觉特征检索（CLIP，图片内容检索）
- **RRF 融合**：RRF_score(doc) = Σ(1 / (k + rank_i))

**效果**：检索准确率提升约 30%（相比传统单一检索）

**面试话术**：
> "我首创了 Dense + Sparse + Visual 三路并行检索与 RRF 融合机制。Dense 捕获语义相似度，Sparse 捕获关键词精确匹配，Visual 捕获视觉特征，通过 RRF 融合互补召回，在复杂查询场景下能更稳定地召回真正相关的片段。"

### 3. 两阶段重排优化 ⭐⭐⭐⭐

**创新**：RRF 粗排 + Cross-Encoder 精排的两阶段重排机制

**实现**：
- **第一阶段**：RRF 粗排，融合多路检索结果，选出 Top-50 候选
- **第二阶段**：Cross-Encoder 精排（BGE-Reranker），精确打分，选出 Top-10

**效果**：Top-K 准确率显著提升（在保证检索效率的同时）

**面试话术**：
> "我设计了两阶段重排机制：先用 RRF 对多路检索结果粗排，再用 Cross-Encoder 对 Top-K 候选精排。这样既保证了检索效率，又显著提升了 Top-K 准确率。"

### 4. One-Pass 意图识别 ⭐⭐⭐⭐

**创新**：将意图分类、查询改写、关键词提取统一为一次 LLM 调用

**传统方案**：
```
意图分类 → 查询改写 → 关键词提取
延迟：200ms + 200ms + 200ms = 600ms
```

**One-Pass 方案**：
```
统一提示词 → 结构化 JSON 输出
延迟：200-500ms（单次调用）
```

**效果**：延迟降低约 60%

**面试话术**：
> "我创新性地将意图分类、查询改写、关键词提取等任务统一为一次 LLM 调用，输出结构化 JSON。相比多轮调用，延迟降低 60%，同时保证了分析质量。"

### 5. 多模态向量化融合 ⭐⭐⭐⭐

**创新**：VLM + CLIP 双路融合策略

**实现**：
- **VLM 路径**：图片 → 描述文本 → 向量（捕获语义信息）
- **CLIP 路径**：图片 → 视觉特征向量（捕获视觉特征）
- **融合策略**：加权平均（0.6 * VLM + 0.4 * CLIP）

**效果**：实现文档和图像的统一向量表示，支持跨模态检索

**面试话术**：
> "我采用 VLM + CLIP 双路融合策略，VLM 捕获语义信息，CLIP 捕获视觉特征，融合后兼顾语义和视觉，实现了文档和图像的统一向量表示，支持跨模态检索。"

### 6. 可视化思考过程 ⭐⭐⭐⭐

**创新**：实时展示 AI 推理链路的思考步骤

**实现**：
- SSE 流式推送思考链（thinking）、引用（citation）、回答（message）
- 前端 ThinkingCapsule 组件实时展示
- 包括意图识别、知识库路由、检索策略等阶段信息

**效果**：可解释性和可调试性大幅提升

**面试话术**：
> "我创新性地实现了 AI 推理链路的实时可视化展示，包括意图识别、知识库路由、检索策略等思考步骤。通过 SSE 流式推送和前端可视化组件，让用户能够理解系统每一步的决策过程，大幅提升了可解释性和可调试性。"

## 🔄 请求处理流程（7 步）

```
[1] Chat API 接收请求 (FastAPI + SSE)
    ↓
[2] One-Pass 意图识别 (IntentProcessor)
    ├─ 意图分类 (factual/comparison/analysis/coding/creative)
    ├─ 查询改写 (refined_query)
    ├─ 关键词提取 (sparse_keywords)
    └─ 视觉需求判断 (needs_visual)
    ↓
[3] 知识库画像路由 (KnowledgeRouter)
    ├─ 查询向量化
    ├─ 在 kb_portraits 中检索 TopN 相似节点 (TopN = 30)
    ├─ 按 KB 聚合打分 (加权投票，位置衰减 α = 0.9)
    └─ 路由决策 (单库/多库/全库，差距阈值 = 0.25)
    ↓
[4] 三路混合检索 (HybridSearchEngine)
    ├─ Dense 检索 (语义向量，BGE Embedding)
    ├─ Sparse 检索 (稀疏向量，BGE-M3)
    └─ Visual 检索 (视觉特征，CLIP，如果需要)
    ↓
[5] 两阶段重排 (Reranker)
    ├─ RRF 粗排 (融合多路检索结果，Top-50)
    └─ Cross-Encoder 精排 (BGE-Reranker，Top-10)
    ↓
[6] 上下文构建 (ContextBuilder)
    ├─ 处理检索结果 (去重、排序、限制数量)
    ├─ 生成引用映射 (UUID → 数字ID，文档 1-N，图片 N+1-M)
    └─ 构建多模态上下文字符串 (≤ 4000 tokens)
    ↓
[7] LLM 生成与流式返回 (GenerationService)
    ├─ 系统提示词选择 (根据意图类型)
    ├─ 多模态格式化 (文档/图片 Type A/B 插槽设计)
    └─ SSE 流式推送 (思考链 + 引用 + 回答)
    ↓
前端展示 (ThinkingCapsule + Citation + Message)
```

### Q2: 知识库画像生成算法是如何工作的？

**回答模板**：

> "知识库画像生成算法分为四个阶段：
> 
> 1. **向量采样**：如果数据量小于 5000，全量提取；否则使用蓄水池采样，按文本和图片的比例分配采样数量。
> 
> 2. **K-Means 聚类**：使用 K-Means 算法对采样向量进行聚类，最优 K 值通过公式 K = sqrt(N/2) 计算，并用轮廓系数评估聚类质量。
> 
> 3. **主题摘要生成**：对每个簇取距离中心最近的 5-10 个样本，回查文本内容（懒加载），然后调用 LLM 生成主题摘要并向量化。
> 
> 4. **画像存储**：将画像存储到 Qdrant 的 kb_portraits collection，包含主题关键词、向量中心点、覆盖领域等信息。
> 
> 画像更新触发机制包括：数据量变化超过 20%、距离上次更新超过 7 天、数据量达到绝对阈值、手动强制更新。"

### Q3: 三路混合检索是如何实现的？为什么能提升准确率？

**回答模板**：

> "三路混合检索包括：
> 
> 1. **Dense 检索**：使用 BGE Embedding 模型将查询和文档向量化（4096 维），通过余弦相似度计算语义相似度。
> 
> 2. **Sparse 检索**：使用 BGE-M3 稀疏向量编码器，将查询和文档编码为稀疏向量（词表ID → 权重），支持关键词精确匹配。
> 
> 3. **Visual 检索**：如果查询涉及图片，使用 CLIP 模型提取视觉特征向量，支持图片内容检索。
> 
> 融合机制采用 RRF（Reciprocal Rank Fusion）算法：RRF_score(doc) = Σ(1 / (k + rank_i))，其中 k = 60。
> 
> 能提升准确率的原因：
> - **互补召回**：Dense 捕获语义相似度，Sparse 捕获关键词精确匹配，Visual 捕获视觉特征，三者互补。
> - **并行检索**：三路检索同时进行，总耗时约等于最慢的那一路。
> - **融合优势**：RRF 综合考虑多路排名，避免单一检索的局限性，在复杂查询场景下能更稳定地召回真正相关的片段。"

### Q7: 项目中遇到的最大技术挑战是什么？如何解决的？

**回答模板**：

> "最大技术挑战是**多知识库路由的准确性问题**。
> 
> **问题分析**：
> - 初始方案：简单相似度匹配，准确率低
> - 原因：未考虑知识库内容分布特征，不同知识库的主题分布差异大
> 
> **解决方案**：
> 1. **引入知识库画像**：
>    - 使用 K-Means 聚类分析知识库内容分布
>    - 使用 LLM 生成主题摘要，形成结构化画像
> 
> 2. **优化路由算法**：
>    - TopN 检索：在 kb_portraits 中检索 TopN 相似节点（TopN = 30）
>    - 加权投票：每个 KB 只取前 K 个最相关节点（K = 5），位置衰减加权（α = 0.9）
>    - 归一化决策：Min-Max 归一化到 [0, 1]，按差距阈值决策（gap = 0.25）
> 
> 3. **效果**：路由准确率显著提升，能够稳定地选择最相关的知识库
> 
> **经验总结**：
> - 需要深入理解业务场景（多知识库的特点）
> - 需要结合传统算法（K-Means）和 LLM（主题摘要）
> - 需要持续优化和迭代（根据实际效果调整参数）"
